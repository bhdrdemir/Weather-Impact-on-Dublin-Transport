Project Documentation

1. Project Overview

This project implements a Data Acquisition and Preprocessing Pipeline that investigates how weather conditions affect public transport ridership in Dublin, Ireland. The system acquires data from multiple sources (REST APIs and flat files), extracts meaningful features, applies transformations, and loads the results into a normalised SQLite database. A web-based frontend provides interactive exploration of the data.

3. Data Acquisition

The pipeline acquires data from four sources using two methods:

API Sources:

fetch_openweather_current() — Calls the OpenWeatherMap REST API to retrieve real-time Dublin weather (temperature, humidity, wind, rain, clouds). Uses free tier (1,000 calls/day). Returns a parsed dictionary with error handling and timeout protection.

fetch_open_meteo_historical() — Calls the Open-Meteo archive API (no key required) to fetch hourly historical weather for Dublin (2018-2025). Implements rate limiting with time.sleep() between yearly chunks. Returns a DataFrame with ~60,000 hourly records.

CSV Sources:

load_met_eireann_daily() — Reads Met Eireann Dublin Airport daily weather CSV. Handles 24 rows of station metadata, cleans string-to-numeric conversions. 30,000+ records from 1942-2026.

load_dublin_bus() / load_luas() — Reads CSO transport statistics. Filters summary rows, combines Luas Red+Green lines into total. Monthly ridership data 2014-2024.

4. Feature Extraction

Feature extraction transforms raw data into meaningful analytical variables. This is a critical step that enables correlation analysis between weather and transport patterns.

5. Transformations

Key transformation functions, each independently testable:

classify_rainfall(rain_mm) — Categorises daily rainfall using Met Eireann meteorological thresholds. Handles NaN and negative values as 'unknown'.

compute_weather_severity(temp, rain, wind) — Weighted composite index combining three normalised components. Rain gets 50% weight as it most directly affects transport decisions (people choosing not to walk to stops/stations).

normalise_passengers(series, method) — Supports min-max (0-1 scaling) and z-score (mean=0, std=1) normalisation. Handles edge cases (constant series, zero standard deviation).

COVID-19 Handling: Years 2020-2021 are flagged with is_covid=True and excluded from all correlation analyses. Lockdown-induced ridership collapse would create spurious weather-transport correlations if included.

6. Database Schema

Data is stored in a normalised SQLite database with 6 tables and indexed for query performance:

weather_daily        — 30,000+ rows: daily weather + extracted features
weather_monthly      — 1,000+ rows: monthly weather aggregates
bus_passengers       — 130+ rows: Dublin Bus monthly ridership + features
luas_passengers      — 80+ rows: Luas monthly ridership + features
weather_transport_merged — 60+ rows: joined weather-transport for analysis
api_weather_log      — API-fetched weather records

Indexes on (year, month) and season columns enable fast filtering in dashboard queries.

7. Testing Strategy

The test suite includes unit tests for each transformation function and an integration test that verifies the full pipeline round-trip:

Unit Tests: classify_rainfall (7 tests), map_season (5 tests), compute_weather_severity (5 tests), normalise_passengers (6 tests), validate_dataframe (4 tests), extract_temporal_features (3 tests), extract_weather_features (3 tests).

Integration Test: Simulates the full pipeline — creates sample data, applies feature extraction and transformations, stores in a test database, queries back using SQL (simulating a frontend request), and validates data integrity across the round-trip. This tests the interaction between frontend (data retrieval) and backend (storage).

8. Ethical Considerations

Data Privacy: All datasets contain aggregated statistics only. No personally identifiable information (PII) is collected or processed.

API Compliance: Both APIs are used within their free-tier limits. Rate limiting (time.sleep()) prevents excessive load on external servers. API keys are stored as configuration variables, not hardcoded.

Data Licensing: All data sources use Creative Commons Attribution 4.0 licences. Full attribution is provided in Section 9.

Bias Awareness: COVID-19 lockdowns created anomalous transport patterns. Excluding 2020-2021 from analysis prevents misleading conclusions while preserving the data for context.
