Project Overview

This project analyzes the impact of weather conditions on public transport usage in Dublin, Ireland. It integrates real-time weather data with historical transport passenger counts to identify correlations, predict demand, and help transport authorities prepare for adverse weather conditions.

The system collects data from five distinct APIs, processes it through a multi-stage ETL pipeline, stores results in SQLite, performs statistical analysis, and provides an interactive web dashboard for exploration and prediction.

Key Outputs: Correlation coefficients showing weather-transport relationships, Kruskal-Wallis tests for group differences, a weather impact simulator, and comprehensive statistical documentation for presentation to stakeholders.

System Architecture

Frontend: Interactive HTML/CSS/JavaScript dashboard with Chart.js visualizations

Backend: Flask REST API with 6 endpoints for data retrieval and pipeline control

Database: SQLite with 8 tables storing weather, transport, and derived features

Data Integration: 5 external APIs for weather and transport data collection

Processing: Multi-stage ETL pipeline with temporal aggregation and feature engineering

Data Acquisition

The system sources data from five distinct APIs, each providing unique insights:

1. Open-Meteo (Historical Weather)

Free weather data service providing historical and forecast data without authentication.

GET /v1/archive?latitude=53.3498&longitude=-6.2603&start_date=2020-01-01&end_date=2024-12-31&hourly=temperature_2m,relative_humidity_2m,precipitation,windspeed_10m

Variables: Temperature, humidity, precipitation, wind speed

2. OpenWeatherMap (Current Weather)

Real-time weather data for Dublin. Requires API key but provides detailed current conditions.

GET /data/2.5/weather?q=Dublin&appid={API_KEY}&units=metric

Variables: Current temperature, conditions, wind, clouds, pressure

3. CSO PxStat REST API (Transport Passengers)

Central Statistics Office Ireland - Official monthly passenger statistics.

GET /PxStat.Products/OpenData/api/v1/jsonstat/dimensions/TDM/TOA14 GET /PxStat.Products/OpenData/api/v1/jsonstat/dimensions/TDM/TOA11

TOA14: Dublin Bus monthly passengers | TOA11: Luas monthly passengers

4. Luas Forecasting API (Real-time Trams)

Real-time XML feed of Luas tram forecasting data for all stops.

GET https://www.luas.ie/forecasting/ASTI/en

Variables: Stop names, due times, destinations, tram directions

5. Irish Rail API (Real-time Trains)

Real-time XML feed of Irish Rail train positions and status.

GET http://api.irishrail.ie/realtime/realtime.asmx/getTrainsXML?TrainType=all

Variables: Train status, location (lat/lon), delays, direction, type

Feature Extraction

Raw data is transformed into engineered features across four categories:

Temporal Features

  • year — Calendar year (2020-2024)
  • month — Month (1-12)
  • day — Day of month (1-31)
  • hour — Hour (0-23)
  • weekday — Day of week (0=Monday, 6=Sunday)
  • is_weekend — Binary: Saturday or Sunday
  • season — Winter, Spring, Summer, Autumn
  • is_rush_hour — Binary: 7-9am or 5-7pm on weekdays

Weather Features

  • rain_category — None, Light, Moderate, Heavy
  • is_rainy — Binary: precipitation > 0.1mm
  • wind_chill — Calculated as: temp - 0.55 × (wind_speed - 4.8)
  • weather_severity — Composite: 25% cold + 50% rain + 25% wind
  • comfort_index — Inverse of severity (0-10 scale)
  • temp_category — Cold, Cool, Mild, Warm, Hot
  • poor_visibility — Binary: cloud_cover > 80%

Transport Features

  • passengers_norm — Z-score normalized passenger count
  • yoy_change — Year-on-year percentage change
  • rolling_avg_3m — 3-month rolling average
  • is_covid — Binary: March 2020 - May 2021

Interaction Features

  • rain_group — Categorical grouping of rain levels
  • temp_group — Categorical grouping of temperatures
  • severity_group — Categorical grouping of weather severity
  • total_passengers — Bus + Luas combined
  • bus_share — Bus / Total passengers (%)
  • weather_impact_score — Predicted passenger impact (0-100)

Data Transformations

Temporal Aggregation

Weather data (hourly from Open-Meteo) is aggregated to daily, then monthly. Transport data arrives monthly from CSO, so alignment is performed by matching year-month pairs.

Merging

Weather monthly aggregates are merged with transport data using outer join on (year, month), forward-filling missing weather values (assumption: stable month weather represents the month).

COVID Exclusion

March 2020 - May 2021 records are marked with is_covid=True and typically excluded from correlation analysis, as lockdowns created anomalous transport patterns.

Normalization & Scaling

Passenger counts are Z-score normalized to enable comparison between bus (larger numbers) and Luas (smaller numbers).

Database Schema

SQLite database with 8 tables storing raw and processed data:

1. weather

Historical monthly weather aggregates from Open-Meteo

Columns: year, month, avg_temp, min_temp, max_temp, avg_humidity, total_rain, avg_wind_speed, avg_pressure, avg_cloud_cover, weather_code

2. transport_bus

Dublin Bus monthly passenger counts from CSO

Columns: year, month, passengers, data_quality

3. transport_luas

Luas monthly passenger counts from CSO

Columns: year, month, passengers, data_quality

4. merged_data

Combined weather + transport with basic features

Columns: year, month, temp, humidity, rain, wind_speed, pressure, cloud_cover, bus_passengers, luas_passengers, weather_code, total_passengers

5. features

Full feature set including temporal, weather, transport, and interaction features

Columns: All features listed in Feature Extraction section

6-8. statistics tables

Pre-computed Pearson, Spearman, and Kruskal-Wallis test results for dashboard display

Statistical Analysis Methods

Pearson Correlation

Measures linear relationship between continuous variables (temperature, rain, wind) and passenger counts.

r = Σ[(x - μx)(y - μy)] / √[Σ(x - μx)² × Σ(y - μy)²]

Interpretation: r ∈ [-1, 1]. Values closer to ±1 indicate strong relationships.

Spearman Rank Correlation

Non-parametric correlation measuring monotonic relationships; less sensitive to outliers.

ρ = 1 - [6Σd² / n(n²-1)] where d = rank difference

Kruskal-Wallis H-Test

Non-parametric test determining if passenger counts differ across categorical weather/time groups.

H = (12 / n(n+1)) × Σ[R²ᵢ / nᵢ] - 3(n+1)

H-statistic compared to χ² distribution; p < 0.05 indicates significant differences.

Simulator: Nearest-Neighbor Prediction

Given test conditions (temp, rain, wind), finds k=5 nearest historical months by Euclidean distance and averages their passenger counts for prediction.

Privacy & Data Use

  • All data sources are official public statistics (CSO) or open APIs (Open-Meteo, Luas, Irish Rail)
  • No personal data collected; analysis is at aggregate (monthly) level only
  • Real-time transport data (Luas, Irish Rail) is public feed; no user tracking

Transparency

  • All methods documented (feature definitions, statistical tests)
  • Code and data transformations available for audit
  • Results include statistical significance (p-values) to prevent overinterpretation

Limitations & Bias

  • COVID period (2020-2021) excluded due to anomalous behavior
  • Correlations do not imply causation; weather is one of many factors affecting transport use
  • CSO data lags by ~2 weeks; real-time Luas/Rail used for current conditions only
  • Analysis assumes 2020-2024 patterns persist; future structural changes may alter relationships
