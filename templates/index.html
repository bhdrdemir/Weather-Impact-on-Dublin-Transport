1. Project Overview

This project implements a full Data Acquisition & Preprocessing Pipeline for analysing how weather affects public transport ridership in Dublin. The deliverable is a local Flask web application that fetches data from three public APIs in real time, runs a feature-extraction pipeline, stores the output in SQLite, and serves an interactive dashboard.

Assignment scope per the B9AI001 CA2 brief: API-based data acquisition, feature extraction, transformation, database loading, unit + integration testing, and documentation — all tied together by a reproducible end-to-end pipeline.

2. System Architecture

User's Browser (index.html, Chart.js) | | HTTP v Flask Backend (app.py) | +-- GET /api/live -> fresh weather + live Luas/Rail (every 60s) +-- POST /api/pipeline/run -> background thread runs full pipeline +-- GET /api/features/sample -> sample rows from every DB table +-- GET /api/dashboard/data, /api/stats -> historical analytics +-- GET /api/tables, /api/query -> DB browser + SQL editor +-- GET /api/tests/run -> unit + integration tests | v Pipeline (pipeline.py) <-> SQLite (dublin_transport.db, 8 tables) | v External APIs: - Open-Meteo (historical + current weather) - CSO PxStat REST (monthly bus + Luas passengers) - Irish Rail (real-time train positions) - Luas Forecasting (real-time tram arrivals - live only)

3. Data Acquisition

Four free public APIs. No API key required for any pipeline step.

Open-Meteo Historical Archive

Hourly weather variables for Dublin. Single call fetches 2022-01-01 to 2024-12-31.

def fetch_open_meteo_historical(start_date, end_date, latitude=53.3498, longitude=-6.2603): url = 'https://archive-api.open-meteo.com/v1/archive' params = { 'latitude': latitude, 'longitude': longitude, 'start_date': start_date, 'end_date': end_date, 'hourly': 'temperature_2m,relative_humidity_2m,rain,' 'wind_speed_10m,surface_pressure,cloud_cover', 'timezone': 'Europe/Dublin' } resp = requests.get(url, params=params, timeout=30) h = resp.json()['hourly'] return pd.DataFrame({ 'timestamp': pd.to_datetime(h['time']), 'temp_c': h['temperature_2m'], 'rain_mm': h['rain'], 'wind_speed_kmh': h['wind_speed_10m'], ... })

CSO PxStat (Ireland's Central Statistics Office)

Monthly passenger counts: TOA14 for Dublin Bus, TOA11 for Luas. Returns JSON-stat 2.0; we flatten its n-dimensional value array into tidy rows.

Weather features

rain_categoryis_rainywind_chillweather_severitycomfort_indextemp_categorypoor_visibility

def compute_weather_severity(temp_c, rain_mm, wind_kmh): """Composite 0-10 index. Cold 25% + rain 50% + wind 25%.""" cold = max(0, (10 - temp_c) / 20) rain = min(rain_mm / 30.0, 1.0) wind = min(wind_kmh / 60.0, 1.0) return round(cold*2.5 + rain*5.0 + wind*2.5, 2)

5. Transformations & Merging

The pipeline joins monthly weather aggregates with monthly CSO passenger counts on (year, month). COVID years (2020-2021) are excluded so the correlation isn't dominated by the lockdown drop.

bus_nc = bus_feat[~bus_feat['is_covid']] # drop COVID years merged = bus_nc.merge(luas_nc, on=['year','month'])\ .merge(weather_monthly, on=['year','month']) merged = extract_merged_features(merged) # add interaction features

6. Database Schema

Eight normalised tables in SQLite:

  • weather_hourly — raw hourly weather + temporal + weather features (~26k rows)
  • weather_daily — daily aggregates (~1.1k rows)
  • weather_monthly — monthly aggregates (~36 rows)
  • bus_passengers — Dublin Bus monthly + transport features
  • luas_passengers — Luas monthly + transport features
  • weather_transport_merged — merged + interaction features (used for correlation)
  • rail_realtime — snapshot of active Irish Rail trains at pipeline run-time
  • luas_realtime — reserved; tram arrivals served live from /api/live instead

8. Statistical Analysis

  • Pearson correlation — linear relationship weather-variable vs passengers
  • Spearman correlation — rank-based, catches monotonic (non-linear) trends
  • Kruskal-Wallis H-test — whether passenger medians differ across weather groups (rain_group, temp_group, severity_group)

All computed on-the-fly from the merged DB table, served via /api/stats.
